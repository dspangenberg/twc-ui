## Features

- Password visibility toggle with secure handling
- Real-time password strength assessment
- Configurable password requirements hint tooltip
- Internationalization support (German/English)
- Accessibility-first design with proper ARIA attributes
- Form validation integration

## Usage

<SourceCode hideHeader={true} code={`import { FormPasswordField } from '@/components/twc-ui/form-password-field'

<FormPasswordField
  name="password"
  label="Password"
  showHint={true}
  showStrength={true}
  isRequired={true}
  onChange={(value) => console.log(value)}
/>`} />

## Props

### Required Props

| Prop | Type | Description |
|------|------|-------------|
| `name` | `string` | The form field name for identification and validation |

### Optional Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | - | Field label displayed above the input |
| `value` | `string \| null` | - | Controlled password value |
| `onChange` | `(value: string \| null) => void` | - | Change handler callback |
| `onBlur` | `() => void` | - | Blur handler callback |
| `description` | `string` | - | Helper text displayed below the field |
| `isRequired` | `boolean` | `false` | Whether the field is required |
| `autoFocus` | `boolean` | `false` | Whether to auto-focus the input |
| `showHint` | `boolean` | `false` | Show password requirements hint tooltip |
| `showStrength` | `boolean` | `false` | Show password strength indicator |
| `placeholder` | `string` | - | Input placeholder text |
| `autoComplete` | `string` | `'current-password'` | Browser autocomplete behavior |

## Password Requirements

When `showHint` is enabled, the component validates passwords against these requirements:

- **Length**: 12 to 24 characters
- **Number**: At least one digit (0-9)
- **Lowercase**: At least one lowercase letter (a-z)
- **Uppercase**: At least one uppercase letter (A-Z)
- **Special**: At least one special character (!@#$%^&*,)(+=._-)

## Password Strength

The strength indicator uses a color-coded system:

- **No input**: Border color (bg-border)
- **Weak (1 requirement)**: Red (bg-red-500)
- **Weak (2 requirements)**: Orange (bg-orange-500)
- **Medium (3 requirements)**: Amber (bg-amber-500)
- **Strong (4+ requirements)**: Emerald (bg-emerald-500)

## Accessibility

- Proper ARIA attributes for form validation
- Screen reader support for requirement status
- Keyboard navigation support
- Focus management for visibility toggle
- Password rules attribute for browser integration

## Integration

The component integrates seamlessly with the form system:

```tsx
import { Form, FormPasswordField } from '@/components/twc-ui'

<Form>
  <FormPasswordField
    name="password"
    label="Password"
    showHint={true}
    showStrength={true}
    isRequired={true}
  />
</Form>
```

## Internationalization

The component supports German and English languages based on the user's locale setting. All labels, requirement descriptions, and strength indicators are automatically translated.

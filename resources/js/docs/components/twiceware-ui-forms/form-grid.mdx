## FormGrid

A responsive grid layout component designed specifically for organizing form fields. It provides a structured way to arrange form elements in columns with optional section titles, borders, and actions.

### Features

- **Flexible Grid System**: Supports 6, 12, or 24 column layouts
- **Responsive Design**: Fields use Tailwind's `col-span-*` classes for width control
- **Section Organization**: Optional titles and borders for form sections
- **Action Support**: Right-aligned action buttons in title area
- **Spacing Control**: Configurable margins and gaps
- **Styling Flexibility**: Customizable title styles and container classes

### Documentation

#### Props

| Prop            | Type                | Default   | Descriptions                                                  |
|-----------------|---------------------|-----------|---------------------------------------------------------------|
| action          | `React.ReactNode`   | ``        | Optional action element (typically buttons) displayed in title area |
| border          | `boolean`           | `false`   | Adds a top border to separate form sections                   |
| children        | `React.ReactNode`   | **Required** | Form fields and content to be displayed in the grid         |
| className       | `string`            | ``        | Additional CSS classes applied to the grid container         |
| cols            | `6 \| 12 \| 24`     | `24`      | Number of columns in the grid system                        |
| fullWidth       | `boolean`           | `true`    | Whether the grid should take full width                       |
| grid            | `boolean`           | `true`    | Whether to apply grid layout (set to false for flex layout)  |
| margin          | `boolean`           | `true`    | Whether to add margin to the grid container                   |
| title           | `string`            | ``        | Section title displayed above the grid                        |
| titleClass      | `string`            | `'font-medium text-sm text-black mt-4 pb-3 border-b'` | CSS classes applied to the title container |

### Grid System

The FormGrid uses a 24-column grid system by default, but can be configured for 6 or 12 columns. Child elements should use Tailwind's `col-span-*` classes to define their width:

- **24-column system**: `col-span-1` to `col-span-24`
- **12-column system**: `col-span-1` to `col-span-12`
- **6-column system**: `col-span-1` to `col-span-6`

### Usage Examples

#### Basic Two-Column Layout

```tsx
import { FormGrid } from '@/components/twc-ui/form-grid'
import { FormTextField } from '@/components/twc-ui/form-text-field'

export function BasicForm() {
  return (
    <FormGrid cols={12}>
      <div className="col-span-6">
        <FormTextField label="First Name" />
      </div>
      <div className="col-span-6">
        <FormTextField label="Last Name" />
      </div>
    </FormGrid>
  )
}
```

#### Non-Grid Layout

```tsx

  return (

```

### Integration with Form Components

FormGrid is designed to work seamlessly with other form components:

```tsx
import { Form, useForm } from '@/components/twc-ui/form'
import { FormGrid } from '@/components/twc-ui/form-grid'
import { FormTextField, FormSelect, FormTextArea } from '@/components/twc-ui/form-text-field'

export function CompleteForm() {
  const form = useForm(/* config */)

  return (
    <Form form={form}>
      <FormGrid title="Contact Information">
        <div className="col-span-12">
          <FormTextField {...form.register('first_name')} label="First Name" />
        </div>
        <div className="col-span-12">
          <FormTextField {...form.register('last_name')} label="Last Name" />
        </div>
        <div className="col-span-24">
          <FormTextField {...form.register('email')} label="Email" />
        </div>
      </FormGrid>
    </Form>
  )
}
```

### Best Practices

1. **Use consistent column systems** throughout your application
2. **Organize related fields** into sections with titles
3. **Use borders** to visually separate major form sections
4. **Keep responsive design** in mind when choosing column spans
5. **Utilize actions** for section-specific buttons (like "Add More" or "Clear")

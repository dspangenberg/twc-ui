## Features

- Built-in stepper buttons for increment/decrement
- Flexible number formatting with Intl.NumberFormatOptions
- Currency, percentage, unit, and decimal support
- Real-time validation and error handling
- Form integration with automatic error display
- Keyboard navigation and accessibility
- Auto-select text on focus
- Configurable min/max values and step increments

## Usage

### Basic Number Field

<SourceCode hideHeader={true} code={`import { Form, FormNumberField, useForm } from '@/components/twc-ui'

const form = useForm('my-form', 'POST', '/submit', {
  quantity: 1,
  price: 0
})

<Form form={form}>
  <FormNumberField
    name="quantity"
    label="Quantity"
    minValue={1}
    maxValue={100}
    description="Number of items to order"
  />
  <FormNumberField
    name="price"
    label="Price"
    formatOptions={{

      currency: 'EUR'
    }}
  />
</Form>`} />

## Props

### Required Props

| Prop | Type | Description |
|------|------|-------------|
| `name` | `string` | The form field name for identification and validation |

### Optional Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | - | Field label displayed above the input |
| `value` | `number \| null \| undefined` | - | Controlled number field value |
| `onChange` | `(value: number \| null) => void` | - | Change handler callback |
| `onBlur` | `() => void` | - | Blur handler callback |
| `description` | `string` | - | Helper text displayed below the field |
| `errorMessage` | `string` | - | Custom error message (overrides form context) |
| `isRequired` | `boolean` | `false` | Whether the field is required (shows red asterisk) |
| `isDisabled` | `boolean` | `false` | Whether the field is disabled |
| `isReadOnly` | `boolean` | `false` | Whether the field is read-only |
| `formatOptions` | `Intl.NumberFormatOptions` | `{style: 'currency', currency: 'EUR'}` | Number formatting options |
| `minValue` | `number` | - | Minimum allowed value |
| `maxValue` | `number` | - | Maximum allowed value |
| `step` | `number` | - | Increment/decrement step value |
| `className` | `string` | - | Additional CSS classes |
| `autoFocus` | `boolean` | `false` | Whether to auto-focus the input |

## Format Options

The component uses `Intl.NumberFormatOptions` for flexible number formatting:

### Currency

<SourceCode hideHeader={true} code={`formatOptions={{
  style: 'currency',
  currency: 'EUR',  // USD, GBP, JPY, etc.
  minimumFractionDigits: 0,
  maximumFractionDigits: 2
}}`} />

### Percentage

<SourceCode hideHeader={true} code={`formatOptions={{
  style: 'percent',
  minimumFractionDigits: 1,
  maximumFractionDigits: 2
}}`} />

### Units

<SourceCode hideHeader={true} code={`formatOptions={{
  style: 'unit',
  unit: 'kilogram',  // celsius, meter, kilometer-per-hour, etc.
  unitDisplay: 'long'  // 'short', 'narrow'
}}`} />

### Decimal

<SourceCode hideHeader={true} code={`formatOptions={{
  style: 'decimal',
  minimumFractionDigits: 2,
  maximumFractionDigits: 4,
  useGrouping: true
}}`} />

## Form Integration

The FormNumberField automatically integrates with your form context:

- **Error Display**: Shows validation errors from the form context
- **Error Variants**: Respects form-level `errorVariant` setting ('form' or 'field')
- **State Management**: Connects to form data and validation state
- **Laravel Integration**: Handles Laravel validation error formatting

## Accessibility

- Proper ARIA attributes for form validation
- Screen reader support for formatted numbers
- Keyboard navigation (arrow keys, page up/down)
- Focus management and auto-selection
- Semantic HTML structure
